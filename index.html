Components

Here we have blocks that are made up of several other elements to form a component. The general rule for a component is that is has a "parent element" with a given class, and the child elements can then have their own styles overwritten.

Jumbotron

Jumbotrons are great

They can draw your eye but they should be used sparingly as they take up a lot of space.

A Jumbotron is fairly basic as components go, it's just a parent .jumbotron that adds the background colour and some padding, then any child elements can be placed inside it. It's useful for title sections or areas where you want to draw attention to something.

Callout

This is a callout, useful for little bits of drama in the middle of a page, or subject dividers, or mini-headers, whatever.

Another simple component, consisting of a .callout with any elements inside it. Recommended usage as above with just a paragraph or 2 of brief text.

Inputs, Textareas, Selects

By default, labels and inputs don't have a fixed width, browser defaults will come into play. However, if you wrap a label/input or label/textarea combo inside a div.input-group the input or textarea will take up as much space as possible up to the label. This can be controlled easily by setting an explicit width on the label itself.

Table

The <table > element should be used to display tabular data only and not for layout or any other use. Therefore we decided to automatically apply styles directly to the table element rather than have a separate class to add these styles.
